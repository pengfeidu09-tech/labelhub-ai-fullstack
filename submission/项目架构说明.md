# 项目架构说明

## 整体架构
LabelHub 采用前后端分离架构，前端 SPA + 后端 REST API。

## 前端架构
```
frontend/src/
  App.tsx              # 路由配置
  main.tsx             # 入口
  layouts/
    MainLayout.tsx     # 侧边栏 + 头部布局
  pages/
    HomePage.tsx       # 首页（角色选择）
    owner/             # 任务负责人页面（11个）
    labeler/           # 标注员页面（4个）
    reviewer/          # 审核员页面（2个）
  components/
    common/            # 通用组件
    designer/          # 模板设计器组件
    renderer/          # 表单渲染器
  api/                 # API 客户端
  stores/              # Zustand 状态
  types/               # TypeScript 类型
  utils/               # 工具函数
```

## 后端架构
```
backend/app/
  main.py              # FastAPI 入口
  core/
    config.py          # 配置（Pydantic Settings）
    database.py        # 数据库连接
    enums.py           # 枚举定义
  models/              # SQLAlchemy 模型
  api/                 # API 路由
  services/            # 业务逻辑
    agent_service.py         # AI Agent 编排
    ai_provider.py           # AI Provider 抽象
    ai_precheck_pipeline.py  # AI 预审流水线
    ai_config_service.py     # 运行时配置
  data/                # 数据文件
```

## 角色体系
- **Owner（任务负责人）**：任务配置、模板搭建、审核规则、结果管理
- **Labeler（标注员）**：任务领取、在线标注、LLM 辅助、草稿提交
- **Reviewer（审核员）**：审核队列、审核详情、通过/打回

## 数据流
1. Owner 创建任务 + 搭建模板 + 配置审核规则 + 发布
2. Labeler 领取 -> 标注 -> LLM 辅助 -> 提交
3. AI Agent 自动预审 -> 多维度评分
4. Reviewer 人工审核 -> 通过/打回
5. 通过数据 -> 结果中心 -> 多格式导出
6. 全链路 -> 审计日志
