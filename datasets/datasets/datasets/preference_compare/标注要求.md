# 标注要求 · 偏好对比标注（preference_compare）

## 一、任务背景

本数据集为「偏好对比标注（Pairwise Preference / RLHF）」题目。每条题目给出同一 **prompt** 下两个模型的回答 **A（response_a）** 与 **B（response_b）**，标注员需判断 **哪个回答更好**，并说明判断依据。结果用于训练奖励模型 / 偏好对齐。

## 二、字段说明

| 字段 | 含义 | 是否参与标注 |
|------|------|-------------|
| `id` | 题目唯一编号 | 否（展示） |
| `task_type` | 任务类型（知识问答 / 代码生成 / 文本摘要 / 翻译 / 创意写作 / 数学推理 / 多轮对话 / 安全合规） | 否（展示） |
| `lang` | 语言或翻译方向 | 否（展示） |
| `prompt` | 用户输入 | 否（展示） |
| `response_a` / `response_b` | **两个待对比的回答（标注核心对象）** | 是 |
| `model_a` / `model_b` | 产出 A/B 的模型来源（标注时建议匿名对待，避免偏向） | 否（展示） |
| `preferred` | **偏好结论：`A` / `B` / `tie`** | 是（填写） |
| `margin` | 优势程度：明显优于 / 略优于 / 相当 | 是（填写） |
| `dimensions` | 做出偏好判断所依据的维度 | 是（填写） |
| `safety_flag` | 是否涉及安全/违规风险（布尔；Excel 中为「是/否」） | 是（核对） |
| `annotator_note` | 标注理由说明 | 是（必填） |

## 三、标注维度（及对应搭建组件）

标注页面由「模板搭建器」用以下物料组件搭建，标注员据此完成偏好判定。下表覆盖课题要求的全部组件类型：

| 标注维度 / 采集项 | 取值 / 说明 | 对应组件 |
|------|------|------|
| **题目原始数据** | 并排渲染 `prompt`、`response_a`、`response_b` 只读，不参与提交 | **展示项 (ShowItem)** |
| **偏好结论** | 三选一：A 更优 / B 更优 / 平局（tie） | **单选** |
| **优势程度（margin）** | 明显优于 / 略优于 / 相当 | **单选** |
| **是否安全风险（safety_flag）** | 是 / 否；任一方含违规内容时置「是」 | **单选** |
| **判断依据维度** | 相关性 / 准确性 / 安全性 / 完整性 / 可读性 / 创意性 / 地道性 等，可多选 | **多选 / 标签选择** |
| **一句话结论** | 简要说明偏好倾向 | **单行输入** |
| **判断理由（annotator_note）** | 指向具体差异点的详细说明（必填） | **多行文本** |
| **改写 / 修订建议** | 对更优回答给出带格式的优化建议 | **富文本编辑器** |
| **结构化批注** | 以结构化形式记录评分明细或更优回答的修订 | **JSON 编辑器** |
| **证据素材** | 上传佐证截图或附件 | **图片上传 / 文件上传** |
| **AI 预判参考** | 调用大模型预测偏好结论与理由，作为标注参考或预填 | **LLM 交互组件** |
