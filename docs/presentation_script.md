# LabelHub 3-Minute Demo Script

## 0:00 - 0:20 项目定位

话术：
LabelHub 是一个面向大模型问答质量评估场景的数据标注与 AI 质量治理平台。它不是单纯 CRUD 标注系统，而是覆盖任务创建、动态标注、AI 预审、人工审核、返修闭环、结果导出和审计追踪的完整数据质量闭环。

画面：展示首页角色选择页面，点击进入 Owner 仪表盘。

## 0:20 - 0:45 Owner 仪表盘

展示内容：
- 项目总览卡片（8项统计指标）
- AI 质量闭环概览（AI平均分、风险分布、一致率）
- 最近业务动态（10条关键动作）
- 推荐演示任务（含跳转按钮）
- 评审演示路径（8步Steps组件）
- 系统健康检查（API/数据库/Demo数据/AI预审/导出服务状态）

话术：
评委打开系统后，可以从首页快速理解任务进度、数据质量、风险分布和演示路径。系统健康检查确保所有组件正常运行。

## 0:45 - 1:15 Labeler 标注工作台

展示内容：
- 三栏布局：左侧原始数据、中间标注表单、右侧AI预审
- Rubric 维度标注（relevance/accuracy/completeness/safety）
- 计时器自动运行
- 保存草稿功能
- AI 预审按钮

话术：
Labeler 在一个三栏工作台中完成数据理解、Rubric 判断、标注填写和 AI 预审，不需要在多个页面来回切换。表单由模板 Schema 动态渲染，支持多种标注场景。

## 1:15 - 1:45 AI 预审 Agent

展示内容：
- overall_score（如 83/100）
- risk_level（如 low/medium/high）
- dimension_scores（4个维度的分项评分）
- suggested_action（如 submit/manual_review/reject）
- reason（质量原因说明）
- 原始 JSON 展示

话术：
AI Agent 会在提交前给出结构化质量建议，帮助提前发现低质量标注、风险样本和完整性问题。当前使用 mock AI Review Agent 保证演示稳定，接口结构支持替换真实模型。

## 1:45 - 2:15 Reviewer 审核详情

展示内容：
- 审核队列列表
- 审核详情页面
- 原始数据展示
- 人工标注结果
- AI 预审结果
- AI / 人工差异对比
- 打回或通过操作

话术：
Reviewer 不只是人工看结果，还可以对比 AI 预审意见，决定通过或打回，形成可追溯的审核闭环。打回时需要填写原因，Labeler 可以看到并修改后重新提交。

## 2:15 - 2:40 结果中心与质量治理

展示内容：
- 统计卡片（总数据量、已提交、已通过、可导出等）
- 质量洞察（AI平均分、风险分布、一致率、重点复核数）
- Rubric 命中分析（4维度统计、高争议标记）
- 重点复核样本（触发原因标签、跳转审核详情）
- AI 质量报告（7节结构化报告、可复制）

话术：
Owner 可以从任务级别查看数据质量，不只是看完成数量，还能看到风险分布、一致率、高争议 Rubric 和重点复核样本。AI 质量报告提供结构化的质量总结和导出建议。

## 2:40 - 3:00 导出与审计

展示内容：
- JSON / CSV / XLSX 导出操作
- 导出记录列表
- 审计日志页面（按动作类型筛选）
- 工时报表

话术：
平台最终能导出可用于后续训练或评测的数据，同时保留全链路审计日志和工时记录，支持复盘和质量追责。从标注到审核到导出，每一步都有迹可循。
