# LabelHub 演示脚本

## 演示概述

本脚本展示 LabelHub 数据标注平台的完整流程，涵盖 Owner、Labeler、Reviewer 三个角色，总时长约 5-10 分钟。

**推荐任务 ID**：
- **Task 4**：完整演示（含已提交、已通过、待审核等多种状态的数据项）
- **Task 1**：AI 审核演示（含 AI 预审结果的数据项）

---

## 演示环境

| 项目 | 值 |
|------|------|
| 前端地址 | http://localhost:5173 |
| 后端地址 | http://localhost:8000 |
| 数据库 | SQLite |
| AI 模式 | Mock Mode（默认，无需 API Key） |

## 演示账号

| 角色 | 登录方式 | 说明 |
|------|----------|------|
| Owner | 任意登录 | 全部权限 |
| Labeler | `labeler_id=2` | 标注员 |
| Reviewer | `reviewer_id=3` | 审核员 |

---

## 演示步骤

### Step 1：Owner 创建任务和模板（1 分钟）

**目标**：展示 Owner 创建任务、配置模板的流程

**操作**：

1. 访问前端首页 http://localhost:5173
2. 进入「任务管理」页面
3. 点击「创建任务」
4. 输入任务名称：`问答质量评估演示任务`
5. 选择模板：`qa_quality`（问答质量评估模板）
6. 启用 AI 审核
7. 保存任务

**讲解要点**：
- 动态模板系统：模板通过 JSON Schema 定义，支持多种标注场景
- AI 审核开关：Owner 可按任务决定是否启用 AI 预审
- 模板设计器：可自定义标注字段、必填规则、评分维度

---

### Step 2：Owner 发布任务（0.5 分钟）

**目标**：展示任务发布流程

**操作**：

1. 在任务列表中找到刚创建的任务（状态为 draft）
2. 点击「发布任务」
3. 确认发布，状态变为 published

**讲解要点**：
- 状态机驱动：draft → published，只有 published 状态的任务才能被标注员领取
- 发布后模板冻结：确保标注期间模板不变更

---

### Step 3：Labeler 领取并标注工作项（1.5 分钟）

**目标**：展示标注员领取任务、在线标注的流程

**操作**：

1. 切换到 Labeler 视角（`labeler_id=2`）
2. 进入「任务广场」，浏览已发布的任务
3. 点击「领取任务」，系统自动分配一条未领取的数据项
4. 进入「标注工作台」，查看待标注数据
5. 查看原始题目（问题 + AI 回答 + 参考答案）
6. 填写标注内容：
   - **相关性**：高相关
   - **准确性**：完全准确
   - **完整性**：完整
   - **安全性**：安全
   - **详细理由**：AI 回答准确覆盖了问题的核心要点，内容完整且无明显错误

**讲解要点**：
- 三级页面导航：任务广场 → 工作台 → 标注表单
- 动态表单渲染：根据模板 Schema 自动渲染标注字段
- 草稿自动保存：标注过程中自动保存，防止数据丢失
- 必填校验：提交前检查必填字段

---

### Step 4：Labeler 触发 AI 预审（1 分钟）

**目标**：展示 AI Agent 自动预审功能

**操作**：

1. 在标注工作台中，点击「AI 预审」按钮
2. 等待 AI 预审完成（Mock 模式即时返回）
3. 查看 AI 预审结果：
   - 综合评分：85
   - 风险等级：low
   - 是否通过：✅ 通过
   - 问题列表：（如有）
   - 质检建议：（如有）

**讲解要点**：
- AI Agent 基于规则引擎实现质量预审
- 结构化输出：评分 + 风险等级 + 结论 + 建议
- Mock 模式：无需 LLM API Key 即可完整演示
- 预审结果持久化：保存到 annotations.json，可追溯

**推荐任务**：使用 Task 1 可查看更多 AI 审核结果示例

---

### Step 5：Labeler 提交标注（0.5 分钟）

**目标**：展示标注提交流程

**操作**：

1. 确认标注内容和 AI 预审结果
2. 点击「提交」按钮
3. 提交成功，状态变为 submitted

**讲解要点**：
- 必填项校验：未填必填字段时提示补充
- 提交后状态变更：draft → submitted
- 提交记录审计日志

---

### Step 6：Reviewer 在审核队列中查看（0.5 分钟）

**目标**：展示审核员的审核队列

**操作**：

1. 切换到 Reviewer 视角（`reviewer_id=3`）
2. 进入「审核中心」
3. 查看待审核列表（状态为 submitted 的标注）

**讲解要点**：
- 审核队列：按提交时间倒序排列
- 显示关键信息：任务名、标注员、提交时间、AI 预审评分

---

### Step 7：Reviewer 审核通过或打回（1 分钟）

**目标**：展示人工审核决策流程

**操作（通过）**：

1. 点击一条待审核记录，进入审核详情
2. 查看原始数据、标注内容、AI 预审结果
3. 点击「通过」
4. 填写审核意见（可选）
5. 确认通过，状态变为 approved

**操作（打回）**：

1. 点击另一条待审核记录
2. 点击「打回」
3. 填写打回原因：`相关性评分需要更详细的理由说明`
4. 确认打回，状态变为 rejected_to_modify

**讲解要点**：
- 三种审核决策：通过 / 打回 / 修订
- 打回后标注员可查看原因并修改
- 审核操作记录审计日志

---

### Step 8：Owner 查看结果中心和质量看板（1 分钟）

**目标**：展示 Owner 视角的结果统计与质量分析

**操作**：

1. 切换到 Owner 视角
2. 进入「结果中心」/「任务详情」
3. 查看任务结果摘要：
   - 总数据项数
   - 已提交数 / 已通过数 / 已打回数
   - AI 审核统计（通过/拒绝/需人工审核）
   - AI 风险分布（low/medium/high）
   - AI 平均评分
   - AI-人工一致性率
   - Top 5 常见问题

**讲解要点**：
- 质量看板：实时展示标注质量指标
- AI-人工一致性：衡量 AI 预审与人工审核的一致程度
- 问题分布：帮助 Owner 发现系统性标注问题

---

### Step 9：Owner 导出数据（1 分钟）

**目标**：展示多格式数据导出功能

**操作**：

1. 在任务详情页点击「导出数据」
2. 选择导出格式：JSON
3. 点击「导出」
4. 等待导出完成
5. 下载文件，查看导出内容
6. 再次导出为 CSV 格式，对比两种格式

**讲解要点**：
- 四种导出格式：JSON / JSONL / CSV / XLSX
- 导出数据包含：原始数据 + 标注结果 + AI 审核结果 + 人工审核结果
- 导出历史：记录所有导出操作
- 审计追踪：每次导出记录审计日志

---

### Step 10：Owner 查看审计日志（0.5 分钟）

**目标**：展示操作审计追踪能力

**操作**：

1. 进入「审计日志」页面
2. 查看操作记录列表：
   - 任务发布记录
   - 数据项领取记录
   - 标注提交记录
   - AI 预审记录
   - 人工审核记录
   - 导出记录
3. 按操作类型筛选（如 `review_approve`）
4. 查看单条记录的详情（操作前/后数据）

**讲解要点**：
- 全链路审计：所有关键状态迁移可追溯
- 操作前/后数据对比
- 支持按用户、操作类型、目标类型筛选
