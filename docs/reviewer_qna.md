# LabelHub Reviewer Q&A

## Q1: 你的项目和普通标注平台有什么区别？

普通标注平台偏任务分发和结果收集，核心能力是"把数据分给人、把结果收回来"。LabelHub 的区别在于：

1. **AI 预审**：标注提交前，AI Review Agent 给出结构化质量建议（总分、风险等级、维度分、建议动作），帮助提前发现低质量标注。
2. **人工复核 + AI/人工差异对比**：Reviewer 审核时可以对比 AI 预审意见和人工标注结果，不只是看一个版本。
3. **返修闭环**：打回后 Labeler 可以看到原因、修改后重新提交，Reviewer 再次审核，形成完整闭环。
4. **质量洞察**：结果中心提供 AI 平均分、风险分布、一致率、Rubric 命中分析、重点复核样本和 AI 质量报告，不只是看完成数量。
5. **全链路审计**：30+ 关键动作追踪，从标注到审核到导出，每一步可追溯。
6. **多格式导出**：JSON / CSV / XLSX，含 AI 预审统计，支持训练数据交付。

## Q2: AI Agent 是否是真实大模型？

当前为了比赛演示稳定，使用 mock AI Review Agent。mock 模式会基于标注内容的完整性和 Rubric 维度给出合理的结构化评分，保证演示效果一致。

但输入输出结构按真实大模型 Agent 设计：
- 输入：原始数据 + 人工标注 + Rubric 维度 + 模板 schema
- 输出：overall_score / risk_level / dimension_scores / suggested_action / confidence / reason / summary / issue_tags

接口设计遵循 Provider 模式，后续只需实现 `ai_provider.py` 即可接入 OpenAI / Claude 等真实模型 API，无需修改业务逻辑。

## Q3: 为什么需要三层嵌套结构？

三层嵌套是 LabelHub 数据模型的核心设计：

**第一层：Project / Task**
承载任务配置、模板关联、AI 开关、状态管理和统计口径。一个 Task 对应一个标注任务。

**第二层：DatasetItem / TaskItem / WorkUnit**
承载具体样本、题目、分配状态和工作单。每个 DatasetItem 包含原始数据（prompt、model_answer、reference 等）。

**第三层：Annotation / Submission / AIReviewRun / HumanReview / WorkSession / ExportRecord / AuditLog**
承载标注结果、提交记录、AI 预审结果、人工审核记录、工作会话、导出记录和审计日志。

为什么这样设计：
- **任务、样本、提交结果解耦**：避免数据冗余，同一任务可以包含不同类型的数据样本
- **支持同一数据多次返修和复审**：每次提交独立追踪，不覆盖历史记录
- **AI 结果与人工结果独立存储**：便于差异对比和持续优化
- **支持全链路审计和导出复盘**：每一步操作可追溯，导出时可关联所有层级信息

## Q4: 如何保证数据质量？

LabelHub 通过多层机制保证数据质量：

1. **AI 预审**：提交前 AI Review Agent 给出质量评分和风险等级，低质量标注可以被提前识别
2. **人工审核**：Reviewer 查看标注详情，对比 AI 预审意见，决定通过或打回
3. **AI/人工差异对比**：审核详情中对比 AI 和人工结果，差异大的样本会被标记
4. **Rubric 命中分析**：4 维度统计（relevance/accuracy/completeness/safety），高争议 Rubric 被标记
5. **重点复核样本**：6 条触发规则筛选（低分、高风险、AI/人工不一致、高争议 Rubric、多次打回、新标注员），自动推荐给 Owner 复核
6. **AI 质量报告**：7 节结构化报告，包含质量摘要、风险分析、导出建议和改进建议
7. **审计日志**：30+ 关键动作追踪，支持复盘和质量追责

## Q5: 导出数据能做什么？

导出的通过数据可以用于：

1. **模型训练**：标注数据作为训练集或微调数据，提升模型在特定任务上的表现
2. **模型评测**：构建评测基准，评估模型在 relevance/accuracy/completeness/safety 等维度的表现
3. **质量分析**：结合 AI 预审统计和人工审核结果，分析数据质量趋势
4. **数据交付**：向业务方交付标注结果，包含原始数据、人工标注、AI 预审和审核信息

导出格式支持 JSON / CSV / XLSX，每种格式包含完整的标注信息和元数据。导出记录保留在平台中，支持追溯。

## Q6: 如果 AI 判断错了怎么办？

AI 只是预审和辅助决策，不替代 Reviewer。具体机制：

1. **AI 建议不等于最终决定**：AI 给出 suggested_action（submit/manual_review/reject），但最终审核由人工完成
2. **AI/人工差异对比**：审核详情中对比 AI 和人工结果，差异大的样本会被标记，Reviewer 可以重点关注
3. **差异记录可追溯**：所有 AI 预审结果和人工审核结果独立存储，便于后续分析 AI 的准确率
4. **持续优化**：积累 AI/人工差异数据后，可以用于优化 AI Agent 的评分逻辑或训练更准确的模型

## Q7: 项目后续如何扩展？

LabelHub 的架构设计支持多个方向扩展：

1. **真实模型 API**：接入 OpenAI / Claude 等真实大模型，替换 mock AI Review Agent
2. **多模态标注**：扩展模板 schema 支持图片、音频、视频标注场景
3. **权限系统**：基于角色的访问控制，支持团队协作和项目隔离
4. **主动学习抽样**：基于 AI 预审结果和模型不确定性，智能推荐标注优先级
5. **模型评测集管理**：将标注数据组织为评测基准，支持模型对比和排行榜
6. **更多导出格式**：JSONL、Parquet 等格式，适配不同训练框架
7. **生产级部署**：Docker Compose / K8s 部署、PostgreSQL 替换 SQLite、Redis 缓存、消息队列
8. **团队协作**：多人标注同一任务、标注一致性统计、标注员能力评估
