# LabelHub 演示流程指南

## 一、项目启动

### 1.1 启动后端

```bash
cd backend
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

后端启动后访问：http://127.0.0.1:8000

### 1.2 启动前端

```bash
cd frontend
npm install  # 首次运行需要安装依赖
npm run dev
```

前端启动后访问：http://localhost:3000

## 二、演示账号说明

| 角色 | ID | 说明 |
|------|-----|------|
| Owner | user_id=1 | 项目所有者，管理任务、模板、数据集和导出 |
| Labeler | labeler_id=2 | 标注员，领取任务并进行标注 |
| Reviewer | reviewer_id=3 | 审核员，审核标注提交结果 |

## 三、完整演示流程

### 第一步：Owner 创建模板

1. 进入 **Owner 端**：点击首页「项目所有者」卡片
2. 进入 **模板管理**：点击左侧菜单「模板管理」
3. 创建问答质量模板：点击「创建问答质量模板」按钮
4. 创建偏好对比模板：点击「创建偏好对比模板」按钮
5. 刷新页面确认模板创建成功

### 第二步：Owner 创建任务

1. 进入 **任务管理**：点击左侧菜单「任务管理」
2. 点击「创建任务」按钮
3. 填写任务信息：
   - 任务名称：如「问答质量评估演示任务」
   - 任务描述：可选
   - 选择模板：选择刚创建的「问答质量模板」
   - 启用AI审核：开启（默认）
4. 点击「创建」完成

### 第三步：Owner 导入演示数据

1. 进入 **任务详情**：在任务列表点击对应任务的「详情」
2. 点击「导入问答质量数据」或「导入偏好对比数据」按钮
3. 确认数据导入成功（数据集列表会显示新数据）

### 第四步：Owner 发布任务

1. 在任务列表或任务详情页面
2. 点击「发布」按钮
3. 确认任务状态变为「已发布」

### 第五步：Labeler 领取任务

1. 进入 **Labeler 端**：点击首页「标注员」卡片
2. 进入 **任务市场**：默认页面
3. 查看已发布任务列表
4. 点击「领取任务」按钮
5. 系统自动跳转到 **标注工作台**

### 第六步：Labeler 进行标注

1. 在标注工作台左侧选择要标注的数据项
2. 填写标注表单：
   - 相关性：选择 High/Medium/Low
   - 准确性：选择 Correct/Partially Correct/Incorrect
   - 完整性：选择 Complete/Partial/Incomplete
   - 安全性：选择 Safe/Risky/Unsafe
   - 总体评价：选择评价等级
   - 详细理由：填写文字说明
3. 点击「💾 保存草稿」保存进度
4. 点击「🚀 提交标注」完成提交

### 第七步：AI Mock 自动审核

提交后，系统会自动触发 AI Mock 审核流程：
- 审核结果会自动更新到提交状态
- 可以在「我的提交」页面查看状态变化

### 第八步：Reviewer 人工审核

1. 进入 **Reviewer 端**：点击首页「审核员」卡片
2. 进入 **审核队列**：默认页面
3. 查看待审核提交列表
4. 点击「审核详情」进入审核详情页
5. 审核操作：
   - 点击「通过」直接通过
   - 点击「打回」填写打回原因
   - 点击「修订」修改标注数据

### 第九步：Owner 导出结果

1. 返回 **Owner 端**
2. 进入 **导出管理**：点击左侧菜单「导出管理」
3. 点击「新建导出」按钮
4. 选择任务和导出格式（json/jsonl/csv/xlsx）
5. 点击「开始导出」
6. 查看导出历史，确认状态变为「成功」

### 第十步：查看审计日志

1. 进入 **审计日志**：点击左侧菜单「审计日志」
2. 查看完整的操作记录链路：
   - 创建任务 → 发布任务
   - 领取数据 → 保存草稿 → 提交标注
   - AI审核开始 → AI审核完成
   - 人工审核通过/打回
   - 创建导出 → 导出完成
3. 点击「查看详情」查看完整的变更数据

## 四、项目亮点

### 4.1 动态模板 Schema

- 支持 JSON Schema 定义表单结构
- 可配置多种数据集类型（问答质量、偏好对比等）
- 模板版本管理

### 4.2 多角色协作

- **Owner**：任务创建者，管理整个标注项目
- **Labeler**：执行标注，贡献标注数据
- **Reviewer**：质量把控，审核标注结果
- 三者协作形成完整的标注工作流

### 4.3 AI Agent Mock 预审

- 模拟 AI 自动审核流程
- 自动判定标注质量
- 提高人工审核效率

### 4.4 状态机流转

完整的生命周期状态管理：
```
草稿 → 已发布 → 已暂停/已结束
数据：待领取 → 已领取 → 已标注
提交：已提交 → AI审核中 → 人工审核中 → 已通过/打回修改
```

### 4.5 审计日志追踪

- 记录所有关键操作
- 追踪数据变更历史
- 支持回溯和问题排查

### 4.6 多格式导出

支持导出格式：
- **JSON**：结构化数据，适合程序处理
- **JSONL**：每行一条记录，适合流式处理
- **CSV**：表格数据，适合数据分析
- **XLSX**：Excel 格式，适合办公使用

## 五、注意事项

1. **演示顺序**：建议按上述流程顺序演示，体现完整链路
2. **数据准备**：首次演示前确保后端已初始化演示数据
3. **状态观察**：注意观察各页面状态变化，理解状态机设计
4. **导出验证**：导出后可在项目目录找到导出文件验证

## 六、快速问答

**Q: 如何清理数据重新演示？**
A: 重启后端服务会自动重置数据库

**Q: 支持自定义模板吗？**
A: 目前支持预置的问答质量和偏好对比模板

**Q: AI 审核是真实的吗？**
A: 当前为 Mock 模式，模拟 AI 审核逻辑

**Q: 导出的文件在哪里？**
A: 默认导出到后端配置的导出目录，可通过 API 获取下载链接
