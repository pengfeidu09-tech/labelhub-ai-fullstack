# LabelHub 演示脚本

## 演示概述

本演示脚本展示 LabelHub 数据标注平台的完整流程，涵盖 Owner、Labeler、Reviewer 三个角色，总时长约 5-10 分钟。

主演示场景：**qa_quality 大模型问答质量标注**
第二演示场景：**preference_compare A/B 偏好对比**

---

## 演示环境

| 项目 | 值 |
|------|------|
| 前端地址 | http://localhost:5173 |
| 后端地址 | http://localhost:8000 |
| 数据库 | SQLite |
| AI 模式 | Mock Mode（默认） |

---

## 演示用户

| 角色 | 用户名 | 密码 |
|------|--------|------|
| Owner | owner | password |
| Labeler | labeler | password |
| Reviewer | reviewer | password |

---

## 演示数据

测试数据位于 `data/` 目录：
- `qa_quality.json` - 问答质量评估数据集
- `preference_compare.json` - A/B 偏好对比数据集

## 演示步骤

### 阶段 1: Owner 创建任务 (1-2 分钟)

**目标**: 展示 Owner 创建任务、配置模板、导入数据、发布任务的流程

#### 1.1 登录 Owner 账户
- 访问前端首页：http://localhost:5173
- 输入用户名 `owner`，密码 `password`
- 点击登录

#### 1.2 创建问答质量评估任务
- 进入「任务管理」页面
- 点击「创建任务」
- 输入任务名称：`问答质量评估任务`
- 选择模板：`qa_quality`（内置模板）
- 启用 AI 审核
- 设置截止日期
- 点击「保存任务」

#### 1.3 导入 qa_quality 测试数据
- 在任务详情页点击「导入数据」
- 选择 `qa_quality.json` 文件
- 点击「导入」
- 确认导入成功（显示导入数量）

#### 1.4 发布任务
- 在任务列表中找到刚创建的任务
- 点击「发布任务」
- 确认发布

---

### 阶段 2: Labeler 标注任务 (2-3 分钟)

**目标**: 展示 Labeler 领取任务、在线标注、自动保存草稿、提交的流程

#### 2.1 切换用户
- 退出当前登录
- 输入用户名 `labeler`，密码 `password`
- 点击登录

#### 2.2 浏览任务广场
- 进入「任务广场」页面
- 查看已发布的任务列表
- 找到「问答质量评估任务」

#### 2.3 领取任务
- 点击「领取任务」
- 确认领取

#### 2.4 开始标注
- 进入「我的任务」页面
- 点击任务进入标注工作台
- 查看待标注数据列表
- 点击第一条数据开始标注

#### 2.5 填写标注内容
- 查看问题和回答内容（Markdown 格式）
- 选择**相关性**：高相关/中等相关/低相关/不相关
- 选择**准确性**：完全准确/部分准确/不准确
- 选择**完整性**：完整/部分完整/不完整
- 选择**安全性**：安全/有风险/不安全
- 填写**详细理由**：请说明判断理由
- 点击「提交」

#### 2.6 继续标注
- 标注剩余数据项（演示 2-3 条即可）

---

### 阶段 3: AI 预审 (自动触发，1 分钟)

**目标**: 展示 AI 自动预审流程，输出结构化评分、结论和质检建议

#### 3.1 查看 AI 审核结果
- 切换到 Owner 账户
- 进入任务详情页
- 查看已提交的标注
- 点击查看 AI 审核结果
- 展示内容：
  - 综合评分：88
  - 结论：pass / reject / human_review
  - 各维度评分详情：相关性、准确性、完整性、安全性、总评
  - 质检建议

---

### 阶段 4: Reviewer 人工审核 (2-3 分钟)

**目标**: 展示 Reviewer 复审流程

#### 4.1 切换用户
- 退出当前登录
- 输入用户名 `reviewer`，密码 `password`
- 点击登录

#### 4.2 查看待审核列表
- 进入「审核中心」页面
- 查看待审核的提交列表

#### 4.3 审核单条提交
- 点击一条提交查看详情
- 查看原始数据、标注内容、AI 审核结果
- 点击「通过」或「打回」或「修订」
- 填写审核意见（如打回需填写原因）

#### 4.4 打回后 Labeler 修改
- 切换到 Labeler 账户
- 查看打回的标注
- 查看打回原因
- 修改标注内容
- 重新提交

---

### 阶段 5: 导出数据 (1 分钟)

**目标**: 展示数据导出功能

#### 5.1 切换用户
- 退出当前登录
- 输入用户名 `owner`，密码 `password`
- 点击登录

#### 5.2 导出任务结果
- 进入任务详情页
- 点击「导出数据」
- 选择导出格式：JSON / JSONL / CSV / Excel
- 点击「导出」
- 下载文件
- 查看导出历史记录

---

## 第二演示：A/B 偏好对比 (2-3 分钟)

**目标**: 展示 preference_compare 场景的标注流程

#### 1. Owner 创建偏好对比任务
- 进入「任务管理」页面
- 点击「创建任务」
- 输入任务名称：`A/B 偏好对比任务`
- 选择模板：`preference_compare`（内置模板）
- 导入 `preference_compare.json` 数据
- 发布任务

#### 2. Labeler 标注偏好对比
- 切换到 Labeler 账户
- 领取「A/B 偏好对比任务」
- 查看并排展示的两个回答（回答 A、回答 B）
- 选择**更优回答**：回答 A / 回答 B / 两者相当
- 选择**差异程度**：明显差异 / 轻微差异 / 无差异
- 选择**安全标记**（如有安全风险）
- 填写**判断理由**
- 提交

#### 3. Reviewer 审核
- 切换到 Reviewer 账户
- 查看偏好对比标注
- 审核通过或打回

## 演示要点

### 主演示 (qa_quality)
1. **模板搭建器**: 展示动态表单配置能力（问答质量评估模板）
2. **实时渲染**: 展示 Designer/Renderer 解耦效果
3. **标注维度**: 相关性、准确性、完整性、安全性、总评、详细理由
4. **AI 审核**: 展示 Mock 模式下的结构化输出（评分、结论、建议）
5. **状态流转**: 展示任务状态机的完整流程
6. **多格式导出**: 展示 JSON/JSONL/CSV/Excel 导出能力

### 第二演示 (preference_compare)
1. **并排展示**: 回答 A、回答 B 并排对比
2. **偏好选择**: preferred（A/B/tie）、margin（明显/轻微/无差异）
3. **安全标记**: safety_flag
4. **判断理由**: 说明选择依据
